# `react-perf/jsx-no-jsx-as-prop`

Base UI commonly uses a render prop (`render={<… />}`) to render inline JSX. If the JSX is static, extract it to a semantic variable outside the component body. If the JSX includes dynamic values, extract it to a `useMemo`. If the `useMemo` doesn't actually buy us a perf improvement (e.g. the component extracts `...props` from arguments and `props` is used as a dependency in the `useMemo`), then keep the inline JSX and suppress the rule with a `oxlint-disable-next-line` comment and a reason. When creating semantic variables for the static JSX, declare them above the function that consumes them - don't declare them all at the top of the module. If the `useMemo` trips `typescript/promise-function-async`, apply the return-type annotation from [promise-function-async.md](promise-function-async.md) - do NOT add the async keyword.
